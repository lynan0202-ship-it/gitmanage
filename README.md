## 🎯 学习路径

### 新手入门
建议按顺序学习前 3 章，掌握 Git 的基本使用。

### 日常开发
重点学习第 4-6 章，掌握撤销、冲突解决和分支管理。

### 团队协作
深入学习第 7-10 章，了解各种工作流和最佳实践。

## 💡 使用建议

1. **理论+实践**：每章学习后都要实际操作练习
2. **循序渐进**：先掌握基础命令，再学习高级技巧
3. **多做笔记**：记录常用命令和遇到的问题
4. **团队规范**：根据团队情况选择合适的工作流
